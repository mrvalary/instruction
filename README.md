## Введение
**Git** — это распределённая система контроля версий. Она позволяет:
- сохранять историю изменений проекта
- работать с ветками
- объединять изменения
- возвращаться к предыдущим версиям файлов
- работать над одним проектом в команде

**GitHub** — это онлайн-сервис для хранения Git-репозиториев и совместной работы над проектами.

### Зачем нужен Git
- **Контроль версий** — вы всегда знаете, кто, когда и зачем внёс изменения
- **Резервное копирование** — код хранится не только на вашем компьютере
- **Совместная работа** — несколько человек могут работать над одним проектом
- **Ветвление** — можно экспериментировать, не ломая основную версию
- **История** — можно откатиться к любому предыдущему состоянию проекта

### Основные понятия

| Термин | Описание |
|--------|----------|
| Репозиторий | Папка проекта, за которой следит Git |
| Коммит | Сохранённое состояние проекта |
| Ветка | Независимая линия разработки |
| Индекс | Область, где подготавливаются файлы перед коммитом |
| Удалённый репозиторий | Копия проекта на сервере (например, GitHub) |
| Клонирование | Создание локальной копии удалённого репозитория |
| Fork | Копия чужого репозитория в вашем аккаунте |

### Как работает Git (кратко)

1. Вы создаёте репозиторий
2. Вносите изменения в файлы
3. Добавляете изменения в индекс (git add)
4. Сохраняете изменения в виде коммита (git commit)
5. Отправляете коммиты на GitHub (git push)
6. Другие участники скачивают изменения (git pull)

## Локальная работа с Git

Локальный Git используется для работы с репозиторием на компьютере пользователя.

### Инициализация репозитория

Команда создаёт новый локальный Git-репозиторий.
``` bash
git init
```

### Проверка состояния репозитория

Команда показывает изменённые, добавленные и неотслеживаемые файлы.
``` bash
git status
```

### Добавление файла в индекс

Команда добавляет файл в область подготовленных изменений.
``` bash
git add имя_файла
```
Пример:
``` bash
git add README.md
```

### Добавление всех файлов

Команда добавляет все изменённые файлы в индекс.
``` bash
git add .
```

### Создание коммита

Коммит фиксирует изменения в истории репозитория.
``` bash
git commit -m "Сообщение коммита"
```
Пример:
``` bash
git commit -m "Добавлена инструкция по Git"
```

### Просмотр истории коммитов

Команда показывает историю изменений.
``` bash
git log
```

### Ключи команды git log

| Ключ | Назначение |
|------|------------|
| --oneline | Каждый коммит в одну строку |
| --graph | История в виде графа веток |
| --all | Коммиты из всех веток |
| --stat | Краткая статистика изменений |
| -p | Подробные изменения в файлах |
| -n 5 | Последние 5 коммитов |
| --author="Имя" | Коммиты конкретного автора |
| --since="2024-01-01" | Коммиты после указанной даты |
| --until="2024-12-31" | Коммиты до указанной даты |
| --grep="текст" | Поиск по тексту сообщения |

Пример:
``` bash
git log --oneline --graph --all
```

### Просмотр изменений

Показывает изменения в файлах, которые ещё не добавлены в индекс.
``` bash
git diff
```
Показывает изменения, которые уже добавлены через git add.
``` bash
git diff --staged
```

### Удаление файлов

Удаляет файл и сразу добавляет это изменение в индекс.
``` bash
git rm file.txt
```
Перестаёт отслеживать файл, но оставляет его на диске.
``` bash
git rm --cached file.txt
```

### Переименование файлов
``` bash
git mv old-name.txt new-name.txt
```

### Отмена изменений

Отменяет незакоммиченные изменения в файле.
``` bash
git restore file.txt
```
Убирает файл из индекса, но сохраняет изменения.
``` bash
git restore --staged file.txt
```

### Создание ветки

Команда создаёт новую ветку.
``` bash
git branch имя_ветки
```
Пример:
``` bash
git branch local
```

### Переключение между ветками

Команда переключает пользователя на другую ветку.
``` bash
git switch имя_ветки
```
Пример:
``` bash
git switch local
```
Альтернативная команда:
``` bash
git checkout local
```

### Просмотр списка веток

Список локальных веток.
``` bash
git branch
```
Список всех веток, включая удалённые.
``` bash
git branch -a
```

### Удаление ветки

Удаляет ветку, если она уже слита.
``` bash
git branch -d branch-name
```
Принудительное удаление.
``` bash
git branch -D branch-name
```
### Временное сохранение изменений

Прячет текущие изменения.
``` bash
git stash
```
Возвращает спрятанные изменения.
``` bash
git stash pop
```

### Пример полного цикла локальной работы
``` bash
git init
git add .
git commit -m "Первый коммит"
git branch develop
git switch develop
echo "Новая функция" > feature.txt
git add feature.txt
git commit -m "Добавлена новая функция"
git switch main
git merge develop
```

## Слияние веток

Слияние используется, чтобы объединить изменения из одной ветки в другую.

### Базовое слияние
``` bash
git checkout main
git merge local
```
### Возможные конфликты

Если одни и те же строки изменены в обеих ветках, Git сообщит о конфликте.

1. Открыть файлы с конфликтами
2. Выбрать правильный вариант вручную
3. Выполнить:
``` bash
git add .
git commit -m "Конфликт разрешён"
```
### Отмена слияния
``` bash
git merge --abort
```

## Краткая памятка команд

| Команда | Назначение |
|---------|------------|
| git init | Создать репозиторий |
| git status | Проверить состояние |
| git add . | Добавить все файлы в индекс |
| git commit -m "..." | Создать коммит |
| git log --oneline | История коммитов |
| git branch | Список веток |
| git switch name | Переключить ветку |
| git merge name | Слить ветку |
| git push origin main | Отправить на GitHub |
| git pull | Загрузить изменения |
| git clone url | Клонировать репозиторий |
