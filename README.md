## Введение

**Git** — это распределённая система контроля версий. Она позволяет сохранять историю изменений проекта, работать с ветками и объединять изменения.

**GitHub** — это онлайн-сервис для хранения Git-репозиториев и совместной работы над проектами.

---

## Создание учётной записи GitHub

Для создания учётной записи GitHub необходимо:

1. Перейти на сайт GitHub
2. Нажать кнопку **Sign up**
3. Ввести email
4. Придумать пароль
5. Указать имя пользователя
6. Подтвердить регистрацию через email
7. После входа в аккаунт создать новый репозиторий

Пример названия репозитория:

git-instruction

## Установка Git

1. Перейдите на официальный сайт: https://git-scm.com
2. Скачайте установщик для вашей операционной системы
3. Запустите установщик
4. Оставьте все настройки по умолчанию (жмите Next)
5. В конце установки поставьте галочку **Git Bash Here**
6. Готово

### Проверка установки

git --version

## Слияние веток

Слияние используется, чтобы объединить изменения из одной ветки в другую.

### Базовое слияние

git checkout main
git merge local

### Возможные конфликты

Если одни и те же строки изменены в обеих ветках, Git сообщит о конфликте.

1. Открыть файлы с конфликтами
2. Выбрать правильный вариант вручную
3. Выполнить:

git add .
git commit -m "Конфликт разрешён"

### Отмена слияния

git merge --abort

## Краткая памятка команд

| Команда | Назначение |
|---------|------------|
| git init | Создать репозиторий |
| git status | Проверить состояние |
| git add . | Добавить все файлы в индекс |
| git commit -m "..." | Создать коммит |
| git log --oneline | История коммитов |
| git branch | Список веток |
| git switch name | Переключить ветку |
| git merge name | Слить ветку |
| git push origin main | Отправить на GitHub |
| git pull | Загрузить изменения |
| git clone url | Клонировать репозиторий |
